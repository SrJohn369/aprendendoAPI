App BLX
App apra anuncio e vendas de produtos na vizinhança

FUNCIONALIDADES:
- Qualquer pessoa poderá anuciar produtos
- Qualquer pessoa podera fazer pedidos anunciados
- Pessoa tem:
	nome
	telefone whatsapp
- Um produto:
	nome
	detalhe
	disponivel
	preço
	foto
- Um pedido:
	pedido
	pessoa
	quantidade
	local de entrega
	entrga ou retirada
	observações
- Cada usuário terá lista de pedidos feitos e recebidos
- O pedido deverá ser aceito pelo vendedor
- O comprador poderá acompanhar seus pedidos:
	- Status

AQUITETURA E FERRAMENTAS
- Python e FastAPI
- Será uma API REST
- Banco de Dados: PostgreSQL e/ou MongoDB (firebase firestore)
- Docker para PostgreSQL
- MVC
- DDD(Domain Driven Design) e Arquitetura limpa (Clean Arch.)


ESTRUTURA DO PROJETO
1. Separação das Preocupações
2. Camadas Principais:
	a. Controller -> Lógica de Negócio -> Percistencia de Dados
3. Benefícios:
	a. Facilidde de Evolução e Manutenção
	b. Trabalho em equipe
	c. Redução de Efeitos Colaterais
	d. Projetos Grandes
	
ORM "Object-Relational Mapping" - Mapeamento Objeto-Relacional
-- É uma técnica de programação que permite aos desenvolvedores trabalharem 
com bancos de dados relacionais de uma forma mais orientada a objetos.

-- Em um banco de dados relacional, os dados são armazenados em tabelas com linhas e colunas, 
enquanto em linguagens de programação orientadas a objetos, os dados são geralmente representados 
como objetos com atributos e métodos. O ORM atua como uma camada intermediária que mapeia automaticamente 
os objetos da linguagem de programação para as tabelas do banco de dados e vice-versa, 
facilitando assim a interação entre o código da aplicação e o banco de dados.
